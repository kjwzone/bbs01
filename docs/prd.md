# prd.md

## 1. 문서 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | Supabase 미니 게시판 |
| 작성 목적 | 초보자가 로그인 기반 게시판을 만들고 배포하기 위한 요구사항 정의 |
| 대상 사용자 | Supabase, Cursor, Vercel 실습 초보자 |
| 기본 기술스택 | Next.js App Router, TypeScript, Supabase, Supabase Auth, Vercel |
| 실습 기준 | Supabase Free Plan |
| 제외 기준 | 댓글, 좋아요, 북마크, 신고, 관리자, 이미지 업로드, Realtime, Playwright MCP, Local Supabase 제외 |

## 2. 제품 개요

| 항목 | 내용 |
|---|---|
| 서비스 한 줄 설명 | 로그인한 사용자가 게시글을 작성하고 본인 글만 수정/삭제할 수 있는 미니 게시판 |
| 해결하려는 문제 | Supabase Auth, DB, RLS, 배포 흐름을 한 번에 실습하기 어렵다 |
| 핵심 가치 | 최소 기능으로 Auth, CRUD, RLS, Vercel 배포 흐름을 경험한다 |
| 주요 사용 상황 | 회원가입 → 로그인 → 게시글 작성 → 본인 글 수정/삭제 → 배포 확인 |

## 3. 대상 사용자

| 사용자 유형 | 설명 | 주요 니즈 | 권한 수준 |
|---|---|---|---|
| 비회원 | 로그인하지 않은 사용자 | 게시글 목록 조회 | 낮음 |
| 회원 | 로그인한 사용자 | 게시글 작성 | 중간 |
| 작성자 | 게시글을 작성한 회원 | 본인 게시글 수정/삭제 | 중간 |

## 4. MVP 기능

| 기능 ID | 기능명 | 설명 | 우선순위 | 포함 여부 |
|---|---|---|---|---|
| MVP-001 | 회원가입 | 이메일/비밀번호로 가입한다 | High | 포함 |
| MVP-002 | 로그인 | 이메일/비밀번호로 로그인한다 | High | 포함 |
| MVP-003 | 로그아웃 | 현재 세션을 종료한다 | High | 포함 |
| MVP-004 | 게시글 목록 조회 | 모든 사용자가 게시글 목록을 볼 수 있다 | High | 포함 |
| MVP-005 | 게시글 작성 | 로그인한 사용자만 게시글을 작성할 수 있다 | High | 포함 |
| MVP-006 | 게시글 수정 | 작성자 본인만 게시글을 수정할 수 있다 | High | 포함 |
| MVP-007 | 게시글 삭제 | 작성자 본인만 게시글을 삭제할 수 있다 | High | 포함 |

## 6. 주요 사용자 흐름

| Flow ID | 사용자 | 시나리오 | 성공 조건 |
|---|---|---|---|
| FLOW-001 | 비회원 | 게시글 목록에 접속한다 | 게시글 목록이 보인다 |
| FLOW-002 | 비회원 | 회원가입한다 | Supabase Auth에 사용자가 생성된다 |
| FLOW-003 | 회원 | 로그인한다 | 현재 사용자 정보가 화면에 표시된다 |
| FLOW-004 | 회원 | 게시글을 작성한다 | posts 테이블에 새 글이 저장된다 |
| FLOW-005 | 작성자 | 본인 글을 수정한다 | 제목 또는 내용이 변경된다 |
| FLOW-006 | 작성자 | 본인 글을 삭제한다 | 게시글이 삭제된다 |
| FLOW-007 | 회원 B | 회원 A의 글을 수정/삭제하려 한다 | RLS 또는 UI에서 차단된다 |
| FLOW-008 | 회원 | 로그아웃한다 | 세션이 종료된다 |

## 7. 권한 요구사항

| 사용자 유형 | 게시글 조회 | 게시글 작성 | 게시글 수정 | 게시글 삭제 | 비고 |
|---|---|---|---|---|---|
| 비회원 | 가능 | 불가 | 불가 | 불가 | 공개 조회만 허용 |
| 회원 | 가능 | 가능 | 본인 글만 가능 | 본인 글만 가능 | RLS 필요 |
| 작성자 | 가능 | 가능 | 본인 글만 가능 | 본인 글만 가능 | posts.author_id 기준 |
